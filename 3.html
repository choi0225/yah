JavaScript

JavaScript이란

자바 스크립트 ( / dʒ ɑː V ə ˌ S K R의 ɪ의 P의 t / ), [8] 종종 약칭 JS는 , A는 하이 레벨 , 해석 프로그래밍 언어 받는 따르는 대한 ECMAScript 사양. JavaScript에는 곱슬 괄호 구문 , 동적 타이핑 , 프로토 타입 기반 객체 지향 및 일류 함수가 있습니다.
HTML 과 CSS 외에도 JavaScript는 World Wide Web 의 핵심 기술 중 하나입니다 . [9] JavaScript는 대화 형 웹 페이지를 가능하게 하며 웹 응용 프로그램 의 필수 부분입니다 . 대다수의 웹 사이트 를 사용, [10] 주요 웹 브라우저 전용이 자바 스크립트 엔진 을 실행합니다.
다중 패러다임 언어 인 JavaScript는 이벤트 중심 , 기능적 및 필수적 ( 객체 지향 및 프로토 타입 기반 포함 ) 프로그래밍 스타일을 지원 합니다. 그것은이 API를 텍스트 작업을위한 배열 , 날짜, 정규 표현식 , 그리고 DOM ,하지만 언어 자체는 포함하지 않는 I / O 등, 네트워킹 , 스토리지 , 또는 그래픽 시설. 이 기능을 제공하기 위해 내장 된 호스트 환경에 의존합니다.
처음에는 웹 브라우저에서 클라이언트 측만 구현되었지만 이제는 JavaScript 엔진이 웹 서버 및 데이터베이스의 서버 측 , 워드 프로세서 및 PDF 소프트웨어 와 같은 웹 이외 프로그램 및 런타임에 포함 된 다른 많은 유형의 호스트 소프트웨어에 포함 됩니다 데스크탑 위젯을 포함하여 모바일 및 데스크톱 응용 프로그램을 작성하는 데 JavaScript를 사용할 수있게 해주는 환경 Vanilla JavaScript 및 Vanilla JS 라는 용어 는 모든 프레임 워크 또는 추가 라이브러리에 의해 확장되지 않는 JavaScript를 의미합니다. 바닐라 JS로 작성된 스크립트는 일반 JavaScript 코드입니다. [11] [12]
언어 이름, 구문 및 해당 표준 라이브러리를 비롯하여 JavaScript와 Java 간에는 유사점이 있지만 두 언어는 구별되며 디자인면에서 크게 다릅니다. JavaScript는 Self 및 Scheme 과 같은 프로그래밍 언어의 영향을 받았습니다 . [13] JSON의 순차 저장하는 형식으로, 데이터 구조 의 파일 또는 가로 질러이를 전송 네트워크는 자바 스크립트 기반으로한다. [14]

역사


1993 년, Urbana-Champaign의 Illinois 대학의 한 부서 인 NCSA ( National Center for Supercomputing Applications ) 는 최초의 유명한 그래픽 웹 브라우저 인 NCSA Mosaic을 출시했습니다. NCSA Mosaic 은 초기 World Wide 의 성장을 확장하는 데 중요한 역할을했습니다. WorldWideWeb 이 3 년 전에 결성 한 NeXTSTEP 틈새 시장을 넘어선 웹 . 1994 년에 Mosaic Communications 라는 회사 가 캘리포니아 주 마운틴 뷰 에 설립되어 원래 NCSA 모자이크 저자의 많은 사람들을 고용하여 Mosaic Netscape. 그러나 의도적으로 NCSA Mosaic과 코드를 공유하지 않았습니다. 이 회사의 브라우저 내부 코드 명은 모질라 (Mozilla)로 , "모자이크와 고질라"의 포루투갈이었다. [15] 웹 브라우저의 첫 번째 버전 인 Mosaic Netscape 0.9는 1994 년 말에 발표되었습니다. 4 개월 이내에 이미 브라우저 시장의 4 분의 3을 차지했으며 1990 년대의 주요 웹 브라우저가되었습니다. NCSA의 상표 소유권 문제를 피하기 위해 브라우저는 같은 해 Netscape Navigator로 이름이 바뀌 었으며 Netscape Communications라는 이름을 사용했습니다. 넷스케이프 커뮤니케이션즈 (Netscape Communications)는 웹이 더욱 다이나믹하게 될 필요가 있음을 깨달았다. 이 회사의 설립자 인 마크 안드레 센 (Marc Andreessen )은 HTML웹 디자이너 및 파트 타임 프로그래머가 이미지 및 플러그인과 같은 구성 요소를 어셈블하는 데 사용하기 쉬운 "접착제 언어"가 필요했습니다.이 코드는 웹 페이지 마크 업에 직접 작성 될 수있었습니다.
1995 년 Netscape Communications 는 Scheme 프로그래밍 언어를 Netscape Navigator 에 임베딩한다는 목표로 Brendan Eich 를 모집 했습니다. [16] 그는 시작도 전에, 넷스케이프 커뮤니케이션과 공동 썬 마이크로 시스템즈 넷스케이프 네비게이터 일의 더 정적 프로그래밍 언어에 포함 자바 와 경쟁하기 위해, 마이크로 소프트 웹 기술 및 플랫폼의 사용자 채택. [17] 넷스케이프는 스크립팅 언어가 자바를 보완 할 만들고 싶었와 같은 다른 언어로 채택 제외 유사한 구문이 있어야한다고 결정 펄 , 파이썬 ,TCL 또는 Scheme. 경쟁 제안에 대해 JavaScript의 아이디어를 방어하기 위해 회사는 프로토 타입이 필요했습니다. Eich는 1995 년 5 월 10 일에 하나를 썼다.
Mocha라는 이름으로 개발되었지만 1995 년 9 월에 Netscape Navigator 2.0의 베타 버전으로 처음 출시되었을 때 공식적으로 LiveScript라고 불렸지 만 12 월에 Netscape Navigator 2.0 베타 3에 배치되었을 때 JavaScript라는 이름이 바뀌 었습니다. [2] [18] 이름의 최종 선택은 언어가 자바 프로그래밍 언어의 스핀 오프, 그리고 선택이 특징되었다는 인상을주는, 혼란의 원인 [19] 넷스케이프에 의해 마케팅 계략는 자바 스크립트를 제공하기로 새로운 웹 프로그래밍 언어였던 것입니다.
Nombas 가 Cmm 이라는 이름으로 개발 한 초기 웹 페이지 스크립팅 언어 ( 1997 년에 작성된 이후 C-- 와 혼동하지 말아야 함)에 의해 JavaScript가 영향을 받았다는 일반적인 오해가 있습니다 . [20] [21] 그러나 Brendan Eich는 LiveScript를 만들기 전에 Cmm에 대해 들어 본 적이 없었습니다. Nombas는 웹 페이지 스크립팅이 ViolaWWW 웹 브라우저 에서 볼 수 있듯이 새로운 개념이 아니지만 내장 웹 페이지 스크립팅을 Netscape에 적용했습니다 . [22] Nombas는 나중에 ScriptEase 제품에서 Cmm 대신 JavaScript를 제공하기로 전환했으며 ECMAScript를 표준화 한 TC39 그룹의 일원이었습니다. [23]
